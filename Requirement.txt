====================================================
BUS TRACKING SYSTEM
System Requirement Document
====================================================

1. Overview
----------------------------------------------------
ระบบนี้เป็นแอปสำหรับติดตามตำแหน่งรถโดยสารแบบ Real-time

วัตถุประสงค์ของระบบ

- ผู้โดยสารสามารถดูตำแหน่งรถในแต่ละสาย
- ผู้ขับรถสามารถเห็นจุดที่มีผู้โดยสารรอ
- ผู้ดูแลระบบสามารถจัดการสายรถ รถ และผู้ใช้งานได้

กลุ่มผู้ใช้งานหลัก

1. Passenger (ผู้โดยสาร)
2. Driver (ผู้ขับรถ)
3. Admin (ผู้ดูแลระบบ)

3. User Roles
====================================================

3.1 Passenger (ผู้โดยสาร)

ผู้โดยสารสามารถ

- ดูรายการสายรถทั้งหมด
- เลือกดูสายรถที่ต้องการ
- ดูรถที่เปิดสถานะ ON ในสาย
- ดูตำแหน่งรถแบบ Real-time บนแผนที่
- กดแจ้งว่ากำลัง "รอรถ"
- ยกเลิกสถานะรอรถได้


3.2 Driver (ผู้ขับรถ)

ผู้ขับรถสามารถ

- Login เข้าระบบ
- เปิดสถานะ ON duty เพื่อเริ่มวิ่งรถ
- ปิดสถานะ OFF duty เพื่อหยุดวิ่งรถ
- ส่งตำแหน่ง GPS ของรถ

เมื่อเปิดสถานะ ON

ระบบจะส่งตำแหน่ง GPS ทุก 5 - 10 วินาที

Driver สามารถเห็น

- เส้นทางของตัวเอง
- จุดที่มีผู้โดยสารรอ
- จำนวนผู้โดยสารที่รอในแต่ละจุด


3.3 Admin (ผู้ดูแลระบบ)

Admin มี 2 ระดับ


Super Admin

สามารถ

- จัดการโซน (Zone) ทั้งหมด
- จัดการเส้นทาง (Route) ทั้งหมด
- จัดการ Admin ทุกคน (สร้าง / แก้ไข / ลบ)
- จัดการคนขับรถ (Driver) ทั้งหมด
- จัดการรถโดยสาร (Bus) ทั้งหมด
- ดูรถทุกสาย
- ดูผู้โดยสารที่รอทั้งหมด
- ดูข้อมูล Analytics ทั้งระบบ


Zone Admin

สามารถ

- เห็นเฉพาะข้อมูลในโซนที่ตัวเองรับผิดชอบ
- จัดการเส้นทางในโซนของตัวเอง
- จัดการคนขับรถในโซนของตัวเอง
- จัดการรถโดยสารในโซนของตัวเอง
- ดูผู้โดยสารที่รอในโซนของตัวเอง

ไม่สามารถ

- จัดการโซนอื่น
- จัดการ Admin คนอื่น
- ดูข้อมูลนอกโซนตัวเอง

13. Admin Dashboard — Flow & ขั้นตอนการใช้งาน
====================================================

13.1 การเข้าสู่ระบบ (Login)
----------------------------------------------------

1. Admin เปิดหน้า Web Dashboard
2. กรอก Username / Password
3. ระบบตรวจสอบ role:
   - super_admin → เข้าถึงข้อมูลทั้งหมด
   - zone_admin  → เห็นเฉพาะข้อมูลในโซนของตัวเอง
4. หลัง Login ระบบแสดงหน้า Dashboard (ภาพรวมสถิติ)


13.2 Dashboard (ภาพรวม)
----------------------------------------------------

แสดงสถิติสรุปของระบบ

- จำนวนโซนทั้งหมด
- จำนวนเส้นทางทั้งหมด
- จำนวนรถโดยสารทั้งหมด
- จำนวนคนขับรถทั้งหมด

Zone Admin จะเห็นเฉพาะสถิติในโซนของตัวเอง


13.3 การจัดการโซน (Zone Management)
----------------------------------------------------

เฉพาะ Super Admin เท่านั้น

ข้อมูลโซน
- zone_code  : รหัสโซน (เช่น CM001)
- zone_name  : ชื่อโซน (เช่น เชียงใหม่)
- status     : สถานะ (active / inactive)

ขั้นตอนสร้างโซน
1. กด "เพิ่มรายการ"
2. กรอก zone_code และ zone_name
3. กด "บันทึก"

ขั้นตอนแก้ไข / ลบ
- กดปุ่ม ✏️ แก้ไข หรือ 🗑️ ลบ ในแถวที่ต้องการ


13.4 การจัดการเส้นทาง (Route Management)
----------------------------------------------------

ข้อมูลเส้นทาง
- route_code     : รหัสเส้นทาง (สร้างอัตโนมัติจาก zone_code)
- route_name     : ชื่อเส้นทาง
- zone_id        : โซนที่เส้นทางนี้สังกัด
- start_location : จุดเริ่มต้น
- end_location   : จุดสิ้นสุด
- status         : สถานะ

ขั้นตอนสร้างเส้นทาง
1. กด "เพิ่มรายการ"
2. เลือกโซน → ระบบสร้าง route_code อัตโนมัติ (กด 🔄 เพื่อสุ่มใหม่)
3. กรอกชื่อเส้นทาง จุดเริ่มต้น จุดสิ้นสุด
4. กด "บันทึก"

Zone Admin: โซนถูกกำหนดอัตโนมัติจากโซนของตัวเอง ไม่สามารถเปลี่ยนได้

การแสดงผล
- จัดกลุ่มตามโซน (ย่อ/ขยายกลุ่มได้)


13.5 การจัดการคนขับรถ (Driver Management)
----------------------------------------------------

ข้อมูลคนขับรถ
- ชื่อ-นามสกุล, Email, เบอร์โทร
- รหัสพนักงาน (สร้างอัตโนมัติเมื่อเลือกเส้นทาง)
- เลขที่ใบขับขี่, ประเภทใบขับขี่
- วันออกบัตร / วันหมดอายุ
- วันเกิด, ที่อยู่, รูปถ่าย
- เส้นทางที่รับผิดชอบ (assigned_route_id)
- ทะเบียนรถที่ผูกอยู่ (แสดงในตาราง)
- สถานะ (active / inactive)

ขั้นตอนสร้างคนขับรถ (2-Step)
1. กด "เพิ่มรายการ"
2. Step 1 — ข้อมูล User Account
   - กรอก Username, Password, ชื่อ, Email, เบอร์โทร
3. Step 2 — ข้อมูลคนขับ
   - เลือกโซน (Super Admin) หรือโซนถูกกำหนดอัตโนมัติ (Zone Admin)
   - เลือกเส้นทาง → รหัสพนักงานสร้างอัตโนมัติ
   - กรอกเลขที่ใบขับขี่ และข้อมูลอื่น ๆ
   - [ไม่บังคับ] กรอกทะเบียนรถ → ระบบสร้างรถโดยสารและผูกคนขับให้อัตโนมัติ
     (สถานะรถที่สร้างอัตโนมัติ = ให้บริการ)
4. กด "บันทึก"

ขั้นตอนแก้ไขคนขับรถ
1. กดปุ่ม ✏️ แก้ไข
2. แก้ไขข้อมูลที่ต้องการ
3. ส่วน "🚍 รถโดยสาร":
   - ถ้ามีรถผูกอยู่แล้ว → แก้ไขทะเบียนได้
   - ถ้ายังไม่มีรถ → กรอกทะเบียนเพื่อสร้างรถใหม่และผูกให้อัตโนมัติ
4. กด "บันทึก"

การแสดงผล
- จัดกลุ่มตามเส้นทาง (ย่อ/ขยายกลุ่มได้)
- คอลัมน์ทะเบียนรถแสดง plate number ของรถที่ผูกกับคนขับ

Reset Password
- กดปุ่ม 🔑 ในแถวคนขับ
- ระบบสร้าง Password ชั่วคราวและแสดงให้ Admin เห็น


13.6 การจัดการรถโดยสาร (Bus Management)
----------------------------------------------------

ข้อมูลรถโดยสาร
- plate_number : ทะเบียนรถ
- driver_id    : คนขับที่ผูกอยู่ (1 คนขับ ต่อ 1 คัน)
- route_id     : เส้นทาง (อ้างอิงจาก assigned_route_id ของคนขับ อัตโนมัติ)
- status       : สถานะ

สถานะรถ
- off         : ไม่ได้ใช้งาน
- on          : ให้บริการ
- maintenance : ซ่อมบำรุง

ขั้นตอนสร้างรถโดยสาร
1. กด "เพิ่มรายการ"
2. กรอกทะเบียนรถ
3. เลือกคนขับ:
   - แสดงเฉพาะคนขับที่ยังไม่ถูกผูกกับรถคันอื่น
   - เส้นทางอ้างอิงจากคนขับที่เลือก (ไม่สามารถแก้ไขเองได้)
4. เลือกสถานะ
5. กด "บันทึก"

ขั้นตอนแก้ไขรถโดยสาร
1. กดปุ่ม ✏️ แก้ไข
2. แก้ไขทะเบียนรถ / เปลี่ยนคนขับ / เปลี่ยนสถานะ
3. เส้นทางอัปเดตอัตโนมัติตามคนขับที่เลือก
4. กด "บันทึก"

การแสดงผล
- จัดกลุ่มตามเส้นทาง (ย่อ/ขยายกลุ่มได้)


13.7 การจัดการ Admin (Admin Management)
----------------------------------------------------

เฉพาะ Super Admin เท่านั้น

ขั้นตอนสร้าง Admin (2-Step)
1. กด "เพิ่มรายการ"
2. Step 1 — ข้อมูล User Account
   - กรอก Username, Password, ชื่อ, Email, เบอร์โทร
3. Step 2 — ข้อมูล Admin
   - เลือกประเภท Admin (zone_admin / super_admin)
   - ถ้าเป็น zone_admin → เลือกโซนที่รับผิดชอบ
4. กด "บันทึก"

ประเภท Admin
- super_admin : เข้าถึงข้อมูลทั้งหมด ไม่มีโซน
- zone_admin  : เข้าถึงเฉพาะโซนที่กำหนด


13.8 การจัดการผู้ใช้งาน (User Management)
----------------------------------------------------

เฉพาะ Super Admin เท่านั้น

- ดูรายการ User ทั้งหมด
- แก้ไขข้อมูล User
- เปลี่ยน Role ของ User
- Reset Password


13.9 รายการรอรับ (Waiting List)
----------------------------------------------------

แสดงรายการผู้โดยสารที่กด "รอรถ" ในระบบ
(อ่านอย่างเดียว ไม่สามารถแก้ไขได้)

ข้อมูลที่แสดง
- ผู้โดยสาร
- เส้นทาง
- ตำแหน่ง (lat, lng)
- เวลาที่รอ
- สถานะ


13.10 หมายเหตุด้านการจัดกลุ่ม (Grouping)
----------------------------------------------------

ตารางที่มีการจัดกลุ่ม (สามารถย่อ/ขยายได้)

- เส้นทาง  → จัดกลุ่มตามโซน
- คนขับรถ  → จัดกลุ่มตามเส้นทาง
- รถโดยสาร → จัดกลุ่มตามเส้นทาง
